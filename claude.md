# TheMatch — астро-совместимость коллег

**Чисто статическая PWA** для расчёта **рабочей** совместимости двух людей по датам
рождения. Фокус — **коллеги и команда**, а не романтика. Движок — только зодиак
(аспектная модель: совместимость = угол между знаками). Весь расчёт идёт **в
браузере** — ни бэкенда, ни базы данных, ни сети. Деплоится как статика (Vercel
preset «Other»), работает оффлайн.

> **Позиционирование.** Это не предсказание и не оценка людей, а **общий язык и
> ice-breaker для команды** — «несерьёзный DISC». Стихии знака переводятся в
> рабочие роли, вывод — actionable («как вам двоим работается»), а не «любовь 87%».
> В UI продукт подаётся как тимбилдинг/фан, НЕ как инструмент найма или оценки.

> История: раньше был Telegram-бот, затем Flask-бэкенд на Vercel, романтическая
> совместимость с тремя блоками (зодиак/биоритмы/нумерология, бленд 35/35/30).
> Бэкенд удалён, логика перенесена в JS. **Биоритмы и нумерология убраны**: как
> независимый шум они схлопывали разброс к среднему и противоречили зодиакальной
> легенде (в half случаев тянули «отличную» пару в серый балл). Продукт
> перефокусирован с романтики на работу. Старый код — в истории git.

> **Важно для Claude:** обновляй этот файл при каждой значимой правке (новые
> файлы/структура, изменения в логике расчёта, деплой). См. «Поддержка файла» внизу.

## Стек

- **Только статика:** HTML + CSS + vanilla JS, без сборки и зависимостей.
- **PWA:** Service Worker (`public/sw.js`, cache-first, оффлайн) + `public/manifest.json`.
- **Деплой:** Vercel как статический сайт. Framework Preset = **Other**, без build
  command. Раздаётся содержимое `public/`. Никаких env-переменных.

## Структура

```
public/             ← всё приложение (статика, раздаётся как сайт)
  index.html        ← разметка: форма ввода двух дат + контейнер результата
  logic.js          ← ВСЯ логика расчёта (зодиак: аспекты, стихии→роли, тексты).
                      Глобальный объект `TheMatch.compute(date1, date2)`
  app.js            ← UI: обработка формы, вызов TheMatch.compute, рендер карточек
  style.css         ← стили, mobile-first (max-width контейнера, тёмная тема)
  sw.js             ← Service Worker (прекэш ассетов, cache-first)
  manifest.json     ← PWA-манифест (standalone, portrait)
  icons/icon.svg    ← иконка приложения
CLAUDE.md
.gitignore
```

## Как это работает

1. Пользователь выбирает две даты рождения (`<input type="date">`) и жмёт кнопку.
2. `app.js` (`handleCalculate`) берёт значения, конвертит ISO → `Date`
   (`isoToDate`) и зовёт **`TheMatch.compute(d1, d2)`** из `logic.js`.
3. `logic.js`:
   - определяет знак каждого → его **стихию** → его **рабочую роль**;
   - считает **аспект** = кратчайшее расстояние между знаками по кругу (0..6) и
     берёт балл из `ASPECT_SCORE`;
   - `total` = балл аспекта. **Симметричен by design** (зависит только от
     расстояния), поэтому `compute(A,B) == compute(B,A)` без усреднений и без
     хранимых матриц 12×12.
4. `app.js` (`renderResults`) рисует hero-кольцо (балл рабочей совместимости),
   роль каждого человека и карточку «как вам работается вместе».

### Источник правды: аспект (расстояние между знаками)

```
ASPECT_SCORE = { 0: 72, 1: 55, 2: 85, 3: 50, 4: 90, 5: 53, 6: 88 }
```

| d | Аспект | Стихии | Рабочая динамика |
|---|--------|--------|------------------|
| 0 | Соединение (тот же знак) | одна | Зеркало: быстро спеваются, но усиливают общие слабости |
| 1 | Полусекстиль (соседи) | разные | Разный темп и язык — нужны явные договорённости |
| 2 | Секстиль | дополняющие | Лучший тандем: дополняют друг друга |
| 3 | Квадрат | конфликт | Трение по умолчанию, но и взаимный рост |
| 4 | Тригон | одна | На одной волне, понимают без слов; риск — эхо-камера |
| 5 | Квинконс | разные | Несостыковка ритмов, договариваться о правилах |
| 6 | Оппозиция | дополняющие | Притяжение противоположностей, сильное взаимодополнение |

### Стихия → рабочая роль (контент-слой)

| Стихия | Знаки | Роль | Сила | Риск |
|--------|-------|------|------|------|
| 🔥 Огонь | Овен, Лев, Стрелец | **Драйвер / Инициатор** | запускает, заряжает, толкает вперёд | не доводит, перегорает, конкуренция за лидерство |
| 🌍 Земля | Телец, Дева, Козерог | **Доводчик / Стабилизатор** | доводит до результата, держит процессы | сопротивляется переменам, медленный старт |
| 💨 Воздух | Близнецы, Весы, Водолей | **Связной / Идеатор** | идеи, коммуникация, нетворк | распыляется, избегает рутины |
| 💧 Вода | Рак, Скорпион, Рыбы | **Клей / Медиатор** | атмосфера, чувствует команду, гасит конфликты | принимает близко к сердцу, избегает прямого спора |

> Per-sign уточнение ролей (напр. Дева — аналитик/QA, Лев — лицо/продажи,
> Козерог — «суровый CEO») — content TODO поверх базовых архетипов стихий.

### Контракт объекта результата (`TheMatch.compute` → `renderResults`)

```
{ total, total_emoji, total_phrase,          // балл рабочей совместимости 0..100
  person1: { sign, sign_name, element, element_emoji,
             role_name, role_strength, role_risk },
  person2: { sign, sign_name, element, element_emoji,
             role_name, role_strength, role_risk },
  pair: { aspect_name,        // соединение/секстиль/тригон/квадрат/оппозиция/...
          dynamic_phrase,     // как им работается вместе
          friction,           // где будет трение
          synergy,            // кто кого дополняет
          tip } }             // один конкретный совет по совместной работе
```

Меняешь форму этого объекта в `logic.js` — синхронно правь рендер в `app.js`.

### Соглашения

- Вся логика и тексты — в `logic.js`; `app.js` только UI и рендер.
- Баллы — числа 0..100; эмодзи/фразы выбираются по диапазонам.
- Тексты на русском, настоящее время, **рабочая рамка** (коллеги/команда/спринт),
  без романтики. Градация позитивная: от «разный язык» к «сильный тандем».
- Никаких ярлыков «токсичный»/«избегайте» на людей — формулировки про *динамику*,
  не про оценку личности (см. позиционирование).

## Деплой (Vercel)

- Подключить репозиторий, **Framework Preset = Other**, Build Command — пусто,
  Output/раздаётся `public/`. Если Vercel не подхватывает `public/` как корень —
  задать **Output Directory = `public`** (или Root Directory = `public`).
- Переменные окружения не нужны. БД/бэкенда нет.

## Локальный запуск

Любой статический сервер из `public/` (Service Worker требует http, не `file://`):

```bash
cd public
python -m http.server 8000   # или: npx serve .
# открыть http://localhost:8000
```
